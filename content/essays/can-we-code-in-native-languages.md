# 我们能用自然语言写代码吗？

# 1

这张照片作为示例图片出现在很多图像识别教材中。照片中的女性名叫Grace Hopper，是美国海军少将，也是计算机科学的先驱之一。

[![](https://img3.doubanio.com/view/thing_review/l/public/p12287442.jpg)](https://img3.doubanio.com/view/thing_review/l/public/p12287442.jpg)

20 世纪 50 年代，她和她领导的团队在为Remington Rand公司工作时，开发了 FLOW-MATIC 编程语言，这是一种接近英语的编程语言，旨在让非程序员，特别是企业经理和其他主管们，能够更容易地理解和阅读程序。

与早期的机器语言或汇编语言不同，FLOW-MATIC 看起来更像是写给人的指令，而不是写给机器的。例如：

> COMPARE A WITH B; IF GREATER GO TO OPERATION 4; IF EQUAL GO TO OPERATION 9; OTHERWISE GO TO OPERATION 6.

这样的语句看起来更像自然语言，而不是传统的编程语法。

然而，尽管 FLOW-MATIC 在形式上接近英语，它仍然要求严格的语法结构和特定的关键词。此外，这种以类英语描述代替数字符号的方法，对于习惯了数学表达式的专业人士来说，反而显得更加啰嗦和晦涩，很多人更倾向于使用Fortran或者Cobol。实际上以一个今天的程序员的视角来看，FLOW-MATIC在功能上显得更像一种表达详尽的汇编语言。

此外，一门语言越接近英语母语者的使用习惯，意味着对于非母语者来说使用它的成本会更高。使用母语编程的愿望，与编程自身的历史一样久远。今天有许多人在试图用自己的母语开发编程语言：如Ramsey Nasser开发了基于阿拉伯语的编程语言alb，；一个名叫Anuu的夏威夷团队试图将C\#进行夏威夷语本地化；美洲原住民后裔程序员Corbett试图回避对笛卡尔坐标系的使用，将它的族人对数学的理解编码进他的艺术项目中。尽管如此，当今世界的主流编程语言仍然是以英语为基础的。

FLOW-MATIC已经消失在历史中了，但这不是Grace Hopper试图连接人类语言与机器语言的唯一尝试，她在1951年参与开发的第一个编译器，为后来出现的许多高级语言打开了大门。

# 2

70 年后，“让机器听懂人说的话”这件事变得更像现实。2025 年初，OpenAI 的联合创始人 Andrej Karpathy 在twitter上说，他最近的工作方式是“氛围编程”（Vibe Coding）：用自然语言告诉模型他想做什么，然后让模型把事情完成。

vibe coding是一种趋势——越来越多的人开始用自然语言编程，人们不需要掌握 Python 或 JavaScript，不需要搞懂变量、循环、回调函数，只需要在 ChatGPT、DeepSeek 或 Claude 这样的界面里打上一句话，比如“我想做一个提醒喝水的小程序”，几秒后，一个完整的代码就出现在屏幕上。

除了对话式模型，还有集成到编辑器里的工具，比如 GitHub Copilot、Cursor、Replit Ghostwriter 等，它们可以在你输入注释时自动生成函数，也能根据代码上下文自动补全整个逻辑。对熟练开发者来说，它像是一个智能助手；对初学者来说，它可能直接取代了学习语法的需求。

今天的大语言模型支持多语种输入。我们可以用中文、法语、韩语描述程序需求，然后让模型翻译成代码。这让更多人能够用自己的母语参与技术开发，不再非得“翻译成英语”才能参与编程。在某种意义上，大语言模型完成了 Grace Hopper 曾经设想过的事情：让人用“自然语言”控制机器。但这一次，自然语言的“自然性”更大程度上依赖于模型的规模、语料的覆盖、语言的主导权。

# 3

23年春天，在毕业和入职之间的间隙里我在上海游荡，认识了改德。他来自青海，是安多地区的藏族人。他从小在寺庙长大，藏语是他生活和学习佛法的第一语言，汉语的学习则是后来在世俗教育系统中逐渐补上的。那段时间我教他汉语和一些现代常识，他教我藏语，并向我普及一些藏传佛教文化。他的汉语水平进步飞速，但我基本只在几个藏语字母中原地踏步。

后来我们都离开了上海，只是偶尔在微信互相问候。最近他问起我的工作，问我在做的工作是不是类似于deepseek的开发，于是我们顺势聊起了这个话题。我好奇他平时是如何使用deepseek的，用藏语还是汉语。他告诉我，deepseek的汉语更丰富自然，使用藏语提问得到的回答往往答非所问，有时甚至无法正确理解他的问题。与其花时间去与模型“解释”藏语，不如直接用他并不熟练但机器更擅长的汉语。虽然有些吃力，但效率更高。

这是一种技术使用中的迁就，也是一种语言选择上的妥协。只要拥有网络，任何人都可以使用这些模型。从 Grace Hopper 所在的年代到今天，不同语言的权力地位确实发生了一定的松动。早期编程语言几乎完全以英语为基础，变量名、结构、语法，都是为英语母语者设计的。如今汉语、法语、韩语等用户也可以直接用自己的母语进行vibe coding，用大模型写故事、生成代码、设计应用。然而对一位藏语母语者来说，即使他拥有一样的技术接入权限，也必须转换语言，才能“正常”使用这些能力。

这种切换不仅仅是语种的切换，也是文化和表达方式的让渡。即便藏语模型已在训练中占据了一席之地，数据仍不够丰富，领域覆盖仍不全面，错误仍比准确多。

我有时会感到一种莫名的愧疚，为我原地踏步的藏语能力和松松垮垮的学习态度。

# 4

我在二十岁之后成为了一名中度听障者，但我的母语仍然是中文口语（普通话和江浙方言）。过去几年里，我艰难地适应着助听器，仍然使用耳朵和嘴巴生活。直到最近我开始学习手语，接触了更多手语聋人，才意识到ta们的语言经验与我完全不同。

手语的表达方式跟口语很不一样，它不依赖声音，而是靠动作、表情、节奏、空间方向。它能表达事实，也能表达情绪、强调、语气和语境。在熟练使用者之间，手语的沟通速度并不比口语慢，甚至有时更高效、更精准。

手语不是“说话的替代品”，它本身就是一种完整的语言，是一些人的母语。而就像口语者打字打得烦了会想要转为更方便的语音输入一样，手语聋人也会在想表达复杂内容时自然地转为手语而不是打字。ta们更愿意用自己的身体直接思考、表达、辩论，而不是被迫将一切翻译成文字。这不是语言能力的缺陷，仅仅是语言类型的不同。

今天主流的大语言模型可以识别语音、生成文本、翻译几十种语言。但如果你对着摄像头打手语，它什么也不会回应。

这是技术发展中的一处巨大盲区。按理说，计算机视觉的发展早于自然语言处理，图像分割、骨架提取、表情识别都已经足够成熟。我们可以通过摄像头识别交警手势、姿态动作、瑜伽体式，却仍没有一个主流AI能让手语聋人像口语者一样自然地“说话”。

许多手语识别项目停留在研究阶段，要么数据集太小，要么只支持单词级识别，无法连贯理解句子。少数能做实时翻译的系统，只能识别美式手语，且准确率远低于语音识别。这并不是一个技术难题，而是一个资源分配问题。你是否能拥有更多输入和表达的便利，取决于你是否属于“多数人”。

技术确实往前走了很多步。从flow-matic到vibe conding，我们用自己的语言与AI对话，编程不再需要熟记语法，甚至不再需要知道代码长什么样。

但“自然语言编程”这个说法本身仍值得追问。它是不是只是一种更易用的接口？是不是仍然只是现实话语权的延伸？“自然语言”这个词不应该只指英语、中文及其他多数人的语言，而应该包括藏语、手语、方言、地方口音，以及一切少数人的表达方式。
